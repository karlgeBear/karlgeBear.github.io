# 一、setup

# 1. 理解

  • setup函数是** Composition API（组合API）的入口**
  • 在setup函数中定义的变量和方法最后都是需要**return**出去的 不然无法再模板中使用(使用<script setup>可以不用return)
  • 创建组件(组件无需注册)实例，然后初始化props，紧接着调用setup函数。它会在beforeCreate, created钩子之前调用。
  • setup返回一个对象。则对象的所有属性(它是响应式的数据)都可以直接在模板中使用。相当于vue2.0中data函数返回的对象。
  • setup(props,context)基本使用
    • props: 组件传值
    • context: 上下文对象，context中还有三个重要的东西attr,slots,emit,他们就相当于vue2中的this.$attrs,this.$slots,this.$emit
<script>
 export default {
  name: 'home',
  setup(){
   let name = 'karlBear'
   let age = 23
   //方法
   function say(){
    console.log(`我叫${name},今年${age}岁`)
   }
   //返回一个对象
   return {
    name,
    age,
    say
   }
  }
 }
</script>

# 二、响应式数据

# 1. ref()和reactive()

  • ref：可传入任意类型的值并返回一个响应式且可改变的ref对象。ref对象拥有一个指向内部值的单一属性.value,改变值的时候必须使用其value属性
  • reactive:接受一个普通对象然后返回该普通对象的响应式代理。等同于2.x的Vue.obserable()

主要使用区别：reactive负责复杂数据结构，ref可以把基本的数据结构包装成响应式

# 2. toRef和toRefs

  • 作用：将响应式对象转换为普通对象，其中结果对象的每个property都是指向原始对象相应的property
  • 场景：从合成函数返回响应式对象时，toRefs非常有用，这样消费组件就可以在不丢失响应式的情况下对返回的对象进行分解/扩散
  • toRef和toRefs的区别
    • toRef是单个转化为响应式，而toRefs是多个转化为响应式
// --------- useToRef.js ---------
import { reactive } from "vue";

export function useToRef() {
    const state = reactive({
        name: 'karlBear',
        age: 23
    })
    // 运行时逻辑抓状态
    
    // 返回时转换为ref
    return state;
}

// --------- App.vue ---------
<script setup>
import {useToRef} from './utils/useToRef'
import {toRefs}  from 'vue'


const state = useToRef();
let toRefsData = toRefs(state)

console.log('响应式数据：',state)
console.log('将响应式数据转化为普通数据：',toRefsData)
</script>

# 3. computed

** 传入一个getter函数，返回一个默认不可修改的ref对象 **

<script setup>
    import {reactive, ref, computed} from 'vue'

    // 1. 响应式变量声明reactive负责复杂 数据结构
    const state = reactive({
        count:  1
    })
    // 2. ref可以把基本的数据结构包装为响应式
    const num =  ref(1)
    // 3. 创建只读的计算属性
    const computedEven1 = computed(() => state.count % 2)
    // 4. 创建可读可写的计算属性
    const computedEven2 = computed({
        get: ()=>{
            return state.count % 2
        },
        set: newVal => {
            state.count = newVal
        }
    })

    // 事件的声明
    function add(){
        state.count++
        num.value += 2 
    }
    // clickEvent
     function handleClick() {
      computedEven2.value = 10;
      console.log('computed创建只可读的计算属性：',computedEven1)
      console.log('computed创建可读可写的计算属性：',computedEven2)
    }
</script>

<template>
    <div>
        <h3>这是reactive传过的数据：{{state.count}}</h3>
        <h3>这是ref传过的数据：{{num}}</h3>
        <button @click="handleClick">点我计算</button>
  </div>
</template>

# 4. watch

  • watch API 完全等效于 Vue2.x this.$watch （以及 watch 中相应的选项）
  • watch接受两个参数
      1. 数据源（要监听的数据）：
      • 一个返回任意值的getter函数
      • 一个包装对象(可以是ref也可以是reactive包装的对象)
      • 一个包含上述两种数据源的数组
      1. 回调函数callBack()（当数据源发生变动时才会被触发）
      1. 可选参数，
      • immediate
      • deep
      • flush：调整函数的刷新时机
      • onTrack / onTrigger : 调试侦听器的依赖
    1. 侦听单个数据源
<script setup>
import {ref, reactive, watch} from 'vue'

//侦听一个reactive定义的数据,修改count值时会触发 watch的回调
const state = reactive({count: 1});
watch(()=>state.count,(newCount,oldCount)=>{
    console.log('----------reactive')
  console.log('newCount:',newCount);  
  console.log('oldCount:',oldCount);
})
//侦听一个ref
const num = ref(0);
watch(num,(newNum,oldNum)=>{
  console.log('----------ref')
  console.log('newNum:',newNum);  
  console.log('oldNum:',oldNum);
})

num.value = 23
state.count = 23
</script>

# 5. watchEffet

  • 作用：立即执行传入的一个函数，并响应式追踪其依赖，并在其依赖变更时重新运行该函数。
  • 特点：默认开启了immediate: true
  • 理解：里边的值改变了一次就调用一次，与computed有点像，只是watchEffect不用写返回值罢了

# 5.1 停止监听

# 隐式监听

当watchEffect在组件的setup()函数或者生命周期钩子被调用时，侦听器会被连接到该组件的生命周期，并在组件卸载时自动停止

# 显示监听
// 直接显示调用返回值来停止侦听
const stop = watchEffet(() => {
	// 
})
stop() // 停止

});
onBeforeUnmount(() => {
    console.log("实例卸载之前");
});
onUnmounted(() => {
    console.log("实例卸载之后");
});
</script>

# 3.1 区别

  1. 在vue2中，我们是先new Vue()，然后执行beforeCreate与created接着问你有没有vm.$mount(el)，有，才继续执行，但是在vue3中，它是先全部准备好后然后再进行函数
  2. 改变了销毁前，和销毁，让它更加语义化了 beforeDestroy改名为beforeUnmount,destroyed改名为unmounted
  3. beforeCreate与created并没有组合式API中,setup就相当于这两个生命周期函数

# 四、hook函数

  • Vue3 的 hook函数 相当于 vue2 的 mixin, 不同在于 hooks 是函数
  • Vue3 的 hook函数 可以帮助我们提高代码的复用性, 让我们能在不同的组件中都利用 hooks 函数
  • 理解：其实就是代码的复用，可以用到外部的数据
//一般都是建一个hooks文件夹，都写在里面
import {reactive,onMounted,onBeforeUnmount} from 'vue'
export default function (){
   //鼠标点击坐标
   let point = reactive({
      x:0,
      y:0
   })

   //实现鼠标点击获取坐标的方法
   function savePoint(event){
      point.x = event.pageX
      point.y = event.pageY
      console.log(event.pageX,event.pageY)
   }

   //实现鼠标点击获取坐标的方法的生命周期钩子
   onMounted(()=>{
      window.addEventListener('click',savePoint)
   })

   onBeforeUnmount(()=>{
      window.removeEventListener('click',savePoint)
   })

   return point
}

# 1. props和emit

  const count = ref(0)
  const increment = () => count.value++
  const decrement = () => count.value--

<script setup>

  const props = defineProps({
    count: {
      type: Number,
      default: 0
    }
  })

  // 声明需要触发的事件，返回一个emit函数，作用和this.$emit函数的作用是一致的
  const emit = defineEmits(['increment', 'decrement'])

# 2. expose和ref

  • expose: 用于声明当组件实例被父组件通过模板引用访问时暴露的公共属性。
// Child.vue
<script setup>
   // 方法一 不适用于Vue3.2版本，该版本 useContext()已废弃
   import { useContext } from "vue"
   const ctx = useContext()
   // 对外暴露属性方法等都可以
   ctx.expose({
       childName: "这是子组件的属性",
       someMethod(){
           console.log("这是子组件的方法")
      }
  })
   
   // 方法二 适用于Vue3.2版本, 不需要引入
   // import { defineExpose } from "vue"
   defineExpose({
       childName: "这是子组件的属性",
       someMethod(){
           console.log("这是子组件的方法")
      }
  })
</script>
 
// Parent.vue 注意 ref="comp"
<template>
   <child ref="comp"></child>
   <button @click="handlerClick">按钮</button>
</template>
<script setup>
   import child from "./child.vue"
   import { ref } from "vue"
   const comp = ref(null)
   const handlerClick = () => {
       console.log(comp.value.childName) // 获取子组件对外暴露的属性
       comp.value.someMethod() // 调用子组件对外暴露的方法
  }
</script>

// Child.vue 接收
<script setup>
   import { defineProps, useContext, useAttrs } from "vue"

       msg1: String

   // 方法二 适用于 Vue3.2版本
   const attrs = useAttrs()
   console.log(attrs) // { msg2:"2222", title: "3333" }
</script>

# 5. provide/inject

  • provide：可以让我们指定想要提供给后代组件的数据
  • inject：在任何后代组件中接收想要添加在这个组件上的数据，不管组件嵌套多深都可以直接拿来用
  • #五、依赖注入(inject 和provide)
// Parent.vue
<script setup>
   import { provide } from "vue"
   provide("name", "karlBear")
</script>
 
// Child.vue
<script setup>
   import { inject } from "vue"
   const name = inject("karlBear")
   console.log(name) // karlBear
</script>

# 6. Vuex

  • store/index.js
// store/index.js
import { createStore } from "vuex"
export default createStore({
   state:{ count: 1 },
   getters:{
       getCount: state => state.count
  },
   mutations:{
       add(state){
           state.count++
      }
  }
})
 
// main.js
import { createApp } from "vue"
import App from "./App.vue"
import store from "./store"
createApp(App).use(store).mount("#app")
 
// Page.vue
// 方法一 直接使用
<template>
   <div>{{ $store.state.count }}</div>
   <button @click="$store.commit('add')">按钮</button>
</template>
 
// 方法二 获取
<script setup>
   import { useStore, computed } from "vuex"
   const store = useStore()
   console.log(store.state.count) // 1
 
   const count = computed(()=>store.state.count) // 响应式，会随着vuex数据改变而改变
   console.log(count) // 1
</script>
